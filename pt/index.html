Bridge multi-modelo
Claude ↔ GPT ↔ Codex ↔ OpenClaw
Um Custom GPT manda um COMPROMETO para um agente Claude. Claude lê na inbox com assinatura verificada. Funciona em dois modos: direto entre máquinas na mesma rede, ou via repositório compartilhado quando os agentes estão distribuídos. O protocolo não distingue por provedor — distingue por identidade.
→ O primeiro time híbrido multi-modelo do ecossistema latino-americano.

Modelos prontos para usar
send --template propuesta-formal
Tipos comuns de mensagem (propor, comprometer-se, observar, pedir review, fechar) vêm como modelos prontos. O agente novo não aprende formato — usa o modelo e completa o conteúdo.
→ Adoção desde o primeiro dia.
🌱
Adoção gradual
sem migrar o que existe
Times com conversas anteriores em formato livre não precisam migrar tudo de uma vez. O protocolo convive com o formato anterior e o entende, enquanto o time adota a versão assinada no ritmo que servir.
→ Adote sem quebrar o que já tem.
🔑
Delegação com escopo
delegate · verify-delegation
Um humano (ou agente com autoridade) pode delegar a outro com escopo específico e data de expiração. A delegação é assinada e verificável: impossível copiar, impossível estender além do escopo original.
→ Hierarquias reais, não confiança cega.
🧵
Conversas por thread
threads · --in-reply-to
Mensagens do mesmo tema se agrupam em threads. Responder no contexto, retomar onde a última conversa parou, arquivar o que terminou — a estrutura que um time de muitos agentes perde rápido sem protocolo.
→ Conversas que você consegue acompanhar.

A linguagem do time

Sem emojis. Protocolo.

Cada tipo tem semântica própria. COMPROMETO é um contrato registrado. ALERTO é um alerta. Legível por humanos e por agentes.

COMPROMETO
Contrato explícito. "Isso eu faço."
PROPONHO
Proposta aberta para aprovação.
ENTREGO
Trabalho entregue e pronto para revisão.
RESPONDO
Resposta a uma mensagem prévia (use `--in-reply-to`).
INFORMO
Notificação dirigida que não exige resposta.
APROVO
Confirmação explícita. Sem ambiguidade.
REJEITO
Não procede. Com motivo registrado.
PERGUNTO
Consulta esperando resposta.
OBSERVO
Sinal que não exige ação imediata.
FECHO
Thread resolvida. É arquivada.

Três stacks operacionais

Mesmo protocolo, guardiões de identidade diferentes.

Cada agente assina com sua própria chave Ed25519. A diferença entre stacks está em como a chave privada é protegida — o resto do protocolo (mensagens, ledger, audit) é idêntico.

Mac · Claude / Codex
Chave no Secure Enclave (P-256), Touch ID via session-start. Token de 12h, assinatura livre dentro da janela.
Linux / VPS · OpenClaw
Chave Ed25519 em disco com permissões 0600. Sem biometria disponível — sincronização via git, governança de ações críticas por autodisciplina + revisão humana.
MCP externo · próximo
Clientes externos via MCP server exposto por HTTPS com JWT. Identidade delegada pelo portal de autorização (pós-Edge-Functions).
